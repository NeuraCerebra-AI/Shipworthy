# ship-workflow-clarity

**The human lens.** Routes and audits whether a person can tell where they are, what object/state they're acting on, what matters now, what action is legitimate next, what will happen, what will **not** happen, and how to recover. It treats obviousness as workflow comprehension, not prettiness.

It maps workflow cartography before judging, routes each flow to archetype overlays (dashboard, CRUD, form, AI copilot, governance, expert tool, mobile, native, creative/game, CLI-like), and — importantly — warns when a "simpler" fix would strip proof, governance, recovery, accessibility paths, expert controls, or productive friction (**harmful-simplification** warnings).

**Use it alone when** you want a clarity/comprehension read on a screen, flow, dashboard, or onboarding. → See [`SKILL.md`](SKILL.md).
