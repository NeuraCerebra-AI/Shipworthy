## Governing acceptance rule

A current full Shipworthy run may choose how to investigate, but it may not
choose whether mandatory completion evidence exists. Small target size, strong
findings, model confidence, or a truthful disclosure does not waive a required
wave, verifier, frontier, recovery, or artifact gate.

## Implementation boundaries

- Change only the four skill-owned instructions, renderer/bundle scripts, and
  repository-only tests or fixtures needed to enforce these demonstrated cases.
- Prefer replacing duplicate wording with one early gate; do not add another
  parallel workflow description.
- Preserve the action-first, human-readable HTML report. Large wave, frontier,
  verifier, and recovery evidence stays collapsed or in bounded JSON.
- Preserve historical legacy import readability, but require explicit
  historical-import identity.
- Do not add a core package, public CLI, database, persistence layer, API,
  daemon, portal, MCP server, provider integration, crawler, or browser.
- Do not write installed skill copies while implementing or testing these
  ledger items.

## Required RED/GREEN scenarios

1. Small target attempts to waive the three-wave minimum.
2. Current full run attempts to render through legacy input.
3. Positive discovery yield is mislabeled exhausted.
4. Verifier cites a nonexistent file or invalid line.
5. Controller attempts to self-repair a rejected verifier certificate.
6. Native upload is unavailable while an authorized alternative remains.
7. Host-required structured orchestration is silently bypassed.
8. HTML conflates findings, evidence debt, and passed records.
9. Coverage-confidence and narrative coverage contradict each other.
10. A raw lane finds an action-like surface that synthesis drops.
11. A hidden/off-route control is incorrectly credited as executed.
12. A promised missing path is mislabeled as merely untested.
13. A build script injects zero-yield and closure without source receipts.
14. Equivalent behavior uses report-only lineage that cannot reconcile.
15. A visual bug lacks a retained screenshot and fails fresh reproduction.
16. HTML says zero NOT_PROVEN while debt cards exist.
17. Passed/Keep receives corrective or tautological fix language.
18. Artifact requirements disagree about the canonical JSON filenames.
19. A benchmark fixture is graded as if it claimed production deployment.
